# LongTEQuant: Long-read Transposable Element Quantification

## About

LongTEQuant is a comprehensive tool for identifying and quantifying transposable element (TE)–derived or TE-associated transcripts from long-read RNA-seq data. It integrates transcript abundance estimation with TE-overlap analysis to classify transcripts by their relationship to TEs, including Gene-alone, TE-alone, and TE–Gene chimeric transcripts.

## Features

- **Integrated Quantification**: Combines long-read and short-read(optional) RNA-seq data for accurate transcript abundance estimation
- **TE–derived transcript identification**: Identifies TE overlaps in transcripts, particularly in first exons and TSS/TES regions
- **Transcript Classification**: Automatically classifies transcripts into Gene-alone, TE-alone, or TE-Gene categories
- **Flexible Analysis**: It is recommended to use miniQuant for quantification, but also supports various quantification tools.
- **Multiple Output Formats**: Generates detailed TSV files with comprehensive TE and transcript information

## Installation

Create conda environment:
```bash
conda env create -f environment.yml
conda activate LongTEQuant
```

A small example dataset is bundled under `test_data/` for sanity checks and demonstrations.  It includes a transcript annotation GTF,a TE annotation GTF, a long‑read SAM file, and a `test.sh` script that runs the core quantification step and produces output identical to the files in `test_data/output/`.  You can use these files immediately after installation to verify the pipeline.


## Input Files

The pipeline requires several input files which must be prepared in advance or provided by the user:

- **Transcript annotation GTF** (`-g`): Transcript annotation file, it is recommended to use the GTF obtained after transcript identification for long-read RNA-seq data (for example, using IsoQuant), but reference gene annotation can also be used.
- **TE annotation GTF** (`-t`): GTF generated by `create_simple_TE_gtf.py` (from RepeatMasker output) containing transposable element coordinates and hierarchy (class, family, subfamily).
- **Long‑read alignment SAM** (`-l`): SAM file of long‑read RNA‑seq alignments (e.g. minimap2 output). Required for overlap counting and miniQuant input.
- **Short‑read alignment SAM** (`-s`, optional): paired short‑read SAM for hybrid quantification. If omitted, the analysis runs in long‑read‑only mode.
- **External quantification TSV** (`-q`, optional for `replace_tpm_from_quant.py`): table from another quantification tool (IsoQuant, Bambu, etc.) containing at least transcript/isoform IDs and TPM values.

## Usage

### Step 1: Prepare TE Annotation

#### `create_simple_TE_gtf.py`
Convert RepeatMasker .out files to GTF format for TE annotations.

**Purpose:**
- Handles RepeatMasker output format
- Generates standardized GTF with TE hierarchy (class/family/subfamily)
- Calculates consensus lengths and truncation proportions

**Command:**
```bash
python create_simple_TE_gtf.py -i repeatmasker.fa.out -o te_annotation.gtf
```

**Parameters:**
- `-i, --input` (required): Input RepeatMasker .out file path
- `-o, --output` (required): Output GTF file path for TE annotations

---

### Step 2: Run Integrated Quantification

#### `integrated_quantification_sam_only.py`
Main script for integrated transcript quantification and TE detection. This is the core analysis step that combines transcript abundance estimation with TE-overlap analysis.

**Purpose:**
- Integrates long-read and short-read RNA-seq data
- Identifies TE overlaps in transcripts, particularly in first exons and TSS/TES regions
- Classifies transcripts into Gene-alone, TE-alone, or TE-Gene categories
- Generates comprehensive output with quantification and TE information

**Command (Full Analysis):**
```bash
python integrated_quantification_sam_only.py \
    -g transcript_annotation.gtf \
    -t te_annotation.gtf \
    -l long_reads.sam \
    -s short_reads.sam \
    -o results/
```

**Parameters:**
- `-g, --gtf-file` (required): Transcript GTF annotation file
- `-t, --te-gtf` (required): TE GTF annotation file (from Step 1)
- `-l, --long-reads` (required): Long-read alignment SAM file
- `-s, --short-reads` (optional): Short-read alignment SAM file for hybrid analysis 
- `-o, --output` (required): Output directory path
- `--skip-quantification` (optional): Skip miniQuant quantification, perform TE analysis only, tpm of all transcripts wlii be 1
- `--miniquant-params` (optional): Additional miniQuant parameters
- `--te-overlap-threshold` (optional): Minimum TE overlap (bp) for classification (default: 10bp)
- `--te-ratio-threshold` (optional): TE proportion threshold for TE-alone classification (default: 80.0%)
- `--te-feature-threshold` (optional): TSS/TES window overlap threshold (default: 50bp)

**Output Files Generated:**
- `transcript_quantification_with_TE.tsv`: Complete transcript quantification with TE overlap information
- `TE_initiated_transcripts.tsv`: Detailed information for TE-associated transcripts

**Notice:**
- if you want to use miniQuant in hybrid mode
- cd /project/liaojianshu/miniquant_TE/miniQuant_old
- wget -qO- https://miniquant.s3.us-east-2.amazonaws.com/pretrained_models.tar.gz | tar xvz

### Step 3: Calculate TE Expression (Optional)

#### `calculate_te_expression.py`
Calculate and aggregate TE expression levels from transcript TPM data at multiple taxonomic levels.

**Purpose:**
- Aggregates TE expression at locus, subfamily, family, and class levels
- Applies configurable overlap proportion thresholds for fine-grained filtering
- Generates separate output files for each aggregation level
- Enables multi-level TE expression analysis and visualization

**Command:**
```bash
python calculate_te_expression.py \
    -t te_annotation.gtf \
    -i results/transcript_quantification_with_TE.tsv \
    -o te_expression_results/ \
    -p 0.5
```

**Parameters:**
- `-t, --te-gtf` (required): TE GTF annotation file
- `-i, --input-tsv` (required): Input transcript quantification TSV file from Step 2
- `-o, --output` (required): Output directory path for TE expression files
- `-p, --overlap-proportion` (optional): Minimum overlap proportion threshold for filtering (default: 0.5)
  - Only includes TE overlap length/ TE copy length ≥ this value

**Output Files Generated:**
- `TE_loci_exp_TPM.tsv`: Individual TE locus expression (highest resolution)
- `TE_subfamily_exp_TPM.tsv`: Subfamily-level aggregation
- `TE_family_exp_TPM.tsv`: Family-level aggregation
- `TE_class_exp_TPM.tsv`: Class-level aggregation (lowest resolution)

---

### Step 4: Replace TPM Values (Optional)

#### `replace_tpm_from_quant.py`
Use results from external quantification tools to replace or update TPM values in the TE analysis output.

**Purpose:**
- Supports integration with other quantification tools (isoQuant, Salmon, etc.)
- Automatically detects column names in quantification files
- Preserves all other TE analysis columns unchanged
- Enables comparison of different quantification methods

**Command:**
```bash
python replace_tpm_from_quant.py \
    -i results/TE_initiated_transcripts.tsv \
    -q external_quantification.tsv \
    -o results/TE_initiated_real_TPM.tsv
```

**Parameters:**
- `-i, --input` (required): Input TSV file from Step 2 (TE_initiated_transcripts.tsv)
- `-q, --quant` (required): External quantification tool output file
- `-o, --output` (required): Output TSV file path with updated TPM values
- `--isoform-col` (optional): Column name for transcript/isoform ID (auto-detected if not specified)
- `--tpm-col` (optional): Column name for TPM values (auto-detected if not specified)

**Supported Formats:**
- isoQuant output
- Salmon quantification
- miniQuant output
- Other tools with standard TSV format (transcript_id/isoform_id + TPM columns)

## Output Files

### Main Output Files

**`transcript_quantification_with_TE.tsv`**

This is the primary output of the quantification step. Each row corresponds to a transcript and contains the following columns:

- `transcript_id`: identifier from the annotation GTF
- `gene_id`: parent gene identifier
- `transcript_TPM`: estimated abundance in TPM (from miniQuant or external quantification)
- `transcript_length`: length of the transcript (bp)
- `num_TEs`: number of TE overlaps detected within the transcript
- `TE_class`: class of the dominant TE (LINE, SINE, LTR, DNA, etc.) or blank if none
- `first_exon_TE_proportion`: fraction of first exon overlapped by any TE (0–100)
- `total_transcript_TE_proportion`: fraction of whole transcript overlapped by TE
- `tss_200_overlap`: number of base pairs of TE overlap within 200 bp after the transcription start site
- `tes_200_overlap`: number of base pairs of TE overlap within 200 bp before the transcription end site
- `transcript_classification`: one of **Gene-alone**, **TE-alone**, **TE-Gene**
- `transcript_sub_classification`: further TE-Gene subtypes (TSS, TES, TE-exonization, etc.)
- `locus`: coordinate of the dominant TE locus (if any)
- `subfamily`, `family`, `class`: hierarchical TE annotation for the dominant copy
- `length`: length of the dominant TE copy (bp)
- `overlap_length`: number of base pairs of overlap between transcript and dominant TE

_Example row (first lines of `test_data/output/transcript_quantification_with_TE.tsv` provided above)._  


**`TE_derived_transcripts.tsv`**

Contains only transcripts classified as having a TE feature with extensive metadata:

- Columns 1–6 duplicate the basic transcript identifiers and TPM/length
- `first_exon_TE_proportion` and `total_transcript_TE_proportion` as above
- `tss_200_overlap`, `tes_200_overlap` as above
- `transcript_classification`, `transcript_sub_classification`
- `TE`: name of the TE 
- `TE_subfamily`, `TE_family`, `TE_class`: taxonomy of initiating TE
- Detailed information for **dominant TE across entire transcript** and **dominant TE in first exon**
  - repeat name, class, family, subfamily, locus, TE length, overlap length, proportion
- `first_exon_length`: length of the first exon (bp)
- `dominant_TE_first_exon_proportion`: fraction of first exon occupied by the dominant TE

_See `test_data/output/TE_derived_transcripts.tsv` for a full column list and example values._


### TE Expression Files (from `calculate_te_expression.py`)

These files aggregate TPM values at different taxonomic levels. Each TSV contains columns tailored to the aggregation (e.g. `locus`, `subfamily`, `family`, `class` plus `TPM`), as described earlier in Step 3.


### Transcript Classification Summary

Transcripts are assigned to one of three broad categories based on TE overlap and location:

- **Gene-alone**: TE overlap ≤ 10 bp
- **TE-alone**: ≥80 % of transcript TE-derived and overlapping TSS window
- **TE-Gene**: hybrid transcripts showing TE overlaps in TSS/TES/first exon or internal exons

Sub-classifications (TSS, TES, TE-exonization) are recorded in the output files and inform downstream filtering.
